Modulo perfil usuario
RF 4.1 El sistema debe permitir visualizar la información del perfil del usuario registrado en Alpha Tours.
RF 4.2 El sistema debe permitir consultar los paquetes turísticos adquiridos, paquetes favoritos, estado de las reservas y la fecha en que se realizó cada compra.
RF 4.3 El sistema debe permitir cancelar reservas de paquetes turísticos realizadas por el usuario.
RF 4.4 El sistema debe permitir visualizar la fecha programada del viaje correspondiente a cada paquete turístico adquirido.

Modelo inventario
RF 2.1 El sistema debe permitir registrar nuevos paquetes turísticos disponibles en Alpha Tours.
RF 2.2 El sistema debe permitir editar la información de los paquetes turísticos existentes.
RF 2.3 El sistema debe permitir eliminar o desactivar paquetes turísticos del sistema.
RF 2.4 El sistema debe permitir registrar la disponibilidad de cupos para cada paquete turístico.
RF 2.5 El sistema debe permitir consultar en tiempo real la disponibilidad de paquetes turístico
RF 2.6 El sistema debe permitir generar alertas cuando la disponibilidad de un paquete turístico sea baja.
RF 2.7 El sistema debe permitir visualizar reportes de paquetes turísticos y reservas realizadas.
RF 2.8 El sistema debe permitir analizar la demanda y disponibilidad de los paquetes turísticos.
RF 2.9 El sistema debe permitir supervisar los movimientos de reservas y cancelaciones.
RF 2.10 El sistema debe permitir apoyar la toma de decisiones mediante reportes y estadísticas del sistema.
